Step 5: Get Your Flattened Contract Source Code

⚠️ Important

You need to download the flattened contract file that contains all the OpenZeppelin imports combined into one file.

Download the flattened contract:

📥 Download KENO-flattened.sol

Then:

  1. Open the downloaded KENO-flattened.sol file in a text editor
  2. Copy ALL the content (Ctrl+A, then Ctrl+C)
  3. Paste it into the "Enter the Solidity Contract Code below" field on BscScan

Step 6: Constructor Arguments (ABI-encoded)

🔑 Critical Step

You need the exact constructor arguments that were used when deploying the contract. These are ABI-encoded addresses for:

  • Team Wallet
  • Treasury Wallet
  • Liquidity Wallet
  • ICO Wallet

To get your constructor arguments:

  1. Go to your contract on BscScan: View Contract
  2. Click on "Contract" tab
  3. Click "More Options" → "Get Constructor Arguments"
  4. Copy the hex string and paste it into the "Constructor Arguments ABI-encoded" field

Or if you know the wallet addresses used during deployment, you can encode them manually using an ABI encoder.

✅ Success!

If successful, you'll see "Contract Source Code Verified" and your contract page will show:

  • ✅ "Read Contract" and "Write Contract" tabs

🔧 Troubleshooting Common Issues

❌ "Compiled bytecode does not match"

  • Double-check compiler version is exactly: v0.8.20+commit.a1b79de6
  • Make sure Optimization is set to "Yes" with 200 runs
  • Verify you're using the flattened contract file

❌ "Constructor arguments invalid"

  • Get the exact constructor arguments from BscScan transaction
  • Make sure the hex string doesn't have "0x" prefix (BscScan removes it automatically)

❌ "SPDX license identifier not provided"

  • Make sure the first line of your contract is: // SPDX-License-Identifier: MIT
