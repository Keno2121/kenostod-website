🔧 Fix BscScan Verification Error

❌ Current Error:

"Unable to find matching Contract Bytecode and ABI"

This means BscScan can't match your source code with the deployed contract.

✅ Most Likely Cause:

Missing Constructor Arguments!

Your KENO contract requires 4 wallet addresses that were used during deployment. You MUST provide these to BscScan.

Step 1: Get Your Constructor Arguments from BscScan

Visit your contract's deployment transaction:

  1. Go to: https://bscscan.com/address/0x65791E0B5Cbac5F40c76cDe31bf4F074D982FD0E
  2. Click on the first transaction (Contract Creation)
  3. Scroll down to "Input Data"
  4. Click "Decode Input Data" button
  5. You'll see 4 wallet addresses:
    • _teamWallet
    • _treasuryWallet
    • _liquidityWallet
    • _icoWallet

Step 2: Encode the Constructor Arguments

Once you have the 4 wallet addresses, you need to encode them.

Option A: BscScan Auto-Encoding (Easiest)

  1. On the BscScan verification page, scroll down to "Constructor Arguments ABI-encoded"
  2. Click "here" to auto-fill from transaction
  3. BscScan will automatically extract and encode the arguments

Option B: Manual Encoding

If auto-fill doesn't work, use this tool:

Go to: https://abi.hashex.org/
Select: "Constructor"
Add 4 parameters of type "address"
Paste your 4 wallet addresses
Click "Encode"
Copy the result (without "0x" prefix)

Step 3: Complete Verification Form

Fill in ALL these fields on BscScan:

Field Value
Contract Address 0x65791E0B5Cbac5F40c76cDe31bf4F074D982FD0E
Compiler Type Solidity (Single file)
Compiler Version v0.8.20+commit.a1b79de6
License Type MIT License (MIT)
Optimization Yes
Runs (Optimizer) 200
Constructor Arguments GET FROM STEP 1 & 2!

Step 4: Paste Contract Code

Use the flattened contract code from: KENO-CONTRACT-FOR-BSCSCAN.txt

Copy all the code and paste it into "Enter the Solidity Contract Code below" field.

🎯 Quick Reference:

  • Contract: View on BscScan

⚠️ Common Mistakes:

  • ❌ Leaving Constructor Arguments field empty
  • ❌ Wrong compiler version (must be exact: v0.8.20+commit.a1b79de6)
  • ❌ Optimization set to "No" when it should be "Yes"
  • ❌ Runs set to wrong number (must be 200)
  • ❌ Including "0x" prefix in constructor arguments (remove it!)
