What Actually Happens
Inside Every Transaction

Most people see the before and after — KENO moved, fee deducted, done. What happens between those two moments is where the enforcement authority lives. This is the sequence at the code level.

01
TRANSACTION SUBMITTED
User signs and broadcasts the transaction
A user or platform submits a transaction to the BSC network. At this moment, nothing has happened yet — the transaction is pending in the mempool. The fee hasn't been deducted. The value hasn't moved. But the enforcement clock has started.
02
CONTRACT ENTRY — ENFORCEMENT GATE
The FeeCollector modifier runs first — before anything else
The moment the transaction reaches the UTL contract, a function modifier runs. This is the enforcement gate. No business logic executes until this gate passes. The modifier checks: Is the fee payment included? Is the sender authorized for this action? Is the amount within protocol bounds?
// Enforcement modifier — runs BEFORE function body
