Executive Summary

The UTL Protocol MetaMask Snap is a transaction intelligence and passive income layer that installs directly into MetaMask. It serves two populations simultaneously: protection seekers (everyday users who want to stop being silently robbed by MEV bots and hidden DeFi fees) and income seekers (KENO token holders who want to earn a share of protocol revenue without changing how they use their wallet).

The Snap fires on every transaction, analyzes MEV risk, estimates extractable value, and shows the user exactly what is about to leave the ecosystem — before they sign. For opted-in participants who hold KENO, the Snap tracks their loyalty tier, cumulative governance weight, staking earnings, and FAL™ (Flash Arbitrage Loan) opportunities — all without leaving MetaMask.

Core Value Proposition
The average DeFi user loses 0.3–2.0% of transaction value to MEV bots, LP spread manipulation, and bridge friction on every interaction — and has no idea it is happening. UTL Protocol is the first Snap that shows this extraction in real time, offers a route to stop it, and then pays the user 60% of every dollar that gets captured instead of extracted.

User Journey — From Install to Earning

1
Install from Snap Store

User finds UTL Protocol in the MetaMask Snap store and clicks Install. Permissions dialog explains what each permission does. User approves. The onInstall handler fires and shows a welcome panel with the tier roadmap and contract addresses.

2
First Transaction — X-Ray Fires

Before the user signs their next swap or transfer, the Fee X-Ray panel appears automatically. Even without opting in, they see MEV risk, estimated slippage loss, and LP spread extraction — data no other Snap surfaces at this level of detail.

3
Activate via dApp

User visits kenostodblockchain.com, connects their wallet. The dApp calls utl_activate with their wallet address. The Snap syncs their existing on-chain profile, sets their tier, and confirms activation with a full onboarding panel.

4
Stake KENO — Start Earning

User stakes KENO on the platform. The dApp calls utl_updateStake which updates the Snap's local state. An in-wallet notification confirms the new tier and multiplier. From this point, every fee captured by the UTL FeeCollector anywhere in the network earns them a proportional share.

5
Tier Advances — Benefits Compound

Every toll payment builds their tier automatically. At 100 KENO toll paid → Silver (5% discount). At 500 → Gold (can submit governance proposals). At 2,000 → Platinum (2x governance weight). At 10,000 → Diamond (advisory seat at T.D.I.R. Foundation). The Snap tracks this in real time.

6
Weekly Report + FAL Alerts

Every Sunday morning, the Snap delivers an in-wallet weekly report: transaction count, fees contributed, KENO earned. Any time a profitable FAL opportunity opens (>0.5% spread), an alert fires immediately. Users can participate in flash arbitrage without leaving the ecosystem.

Security & Privacy Disclosure

No Keys. Ever.
The Snap never requests, stores, or transmits private keys or seed phrases. All state is stored in MetaMask's encrypted snap state using snap_manageState. The only wallet identifier used is the user-provided wallet address for API profile lookups.
API Calls — Read Only
All network requests are GET reads only. No transaction signing happens inside the Snap. No funds move without the user explicitly signing in MetaMask. API calls timeout after 5 seconds with full graceful fallback to locally stored state.
Sandbox Isolated
All Snap logic runs in the MetaMask Snap sandboxed environment. The Snap cannot access other Snaps, other dApps, browser history, or system resources. All computation is deterministic and auditable from the open-source repository.
No Tracking
No analytics SDK is bundled. No user behavior is reported to third-party services. The wallet address used for API profile sync is provided by the user via the dApp opt-in flow — the Snap does not extract it independently.
