Build on UTL Protocol

UTL Protocol is open blockchain infrastructure that any platform can use to distribute BEP-20 tokens based on merit, achievement, or educational completion — with built-in staking, fee collection, and treasury management. All 5 contracts are live on BSC mainnet. Kenostod Blockchain Academy is the first platform built on it. You can be next.

Protocol, Not Product

Just as Uniswap is built on Ethereum, Kenostod is built on UTL Protocol. The protocol itself is open — any platform, any use case, any token reward model. Kenostod is the first implementation. You can build the second, third, or tenth.

Revenue Sharing

Every platform that integrates UTL Protocol generates fee revenue through the FeeCollector contract. A portion flows back to KENO stakers and the T.D.I.R. Foundation. You build. The protocol rewards everyone in the ecosystem.

Key Functions

The most commonly used functions when integrating UTL Protocol into your platform.

Distribution Contract — Send KENO to a user

Solidity 
// Called when a user completes a course, module, or achievement
// Only callable by authorized platform addresses
function distributeTokens(
    address recipient,     // User's wallet address
    uint256 amount,        // Amount of KENO in wei (250 KENO = 250e18)
    string calldata reason // e.g. "Course 1 Completion"
) external returns (bool)

Staking Contract — Check staking balance

Solidity 
// Returns the amount of KENO a user has staked
function stakedBalance(address user) external view returns (uint256)

// Returns pending staking rewards for a user
function pendingRewards(address user) external view returns (uint256)

Quickstart Integration Guide

Get your platform distributing KENO tokens in 4 steps. This covers the most common integration pattern — trigger token distribution when a user completes a defined action.

1

Contact UTL Protocol to get authorized

The Distribution contract only accepts calls from authorized platform addresses. Send your platform's deployer wallet address to kenostod21@gmail.com with subject line "UTL Protocol Integration Request." We'll whitelist your address within 48 hours.

2

Install ethers.js or web3.js

bash 
npm install ethers
3

Initialize the Distribution contract

JavaScript 
import { ethers } from 'ethers';

// UTL Protocol — Distribution Contract
const UTL_DISTRIBUTION = '0xE6918cdBB9D8cd0d3532A88D974734B2F1A793c7';
const KENO_TOKEN      = '0x65791E0B5Cbac5F40c76cDe31bf4F074D982FD0E';

const ABI = [
  'function distributeTokens(address recipient, uint256 amount, string reason) returns (bool)',
  'function stakedBalance(address user) view returns (uint256)',
];

const provider = new ethers.JsonRpcProvider('https://bsc-dataseed.binance.org/');
const signer   = new ethers.Wallet(process.env.PLATFORM_PRIVATE_KEY, provider);
const utl      = new ethers.Contract(UTL_DISTRIBUTION, ABI, signer);
4

Trigger distribution on completion events

JavaScript 
// Call this when your user completes a course, module, or task
async function rewardUser(walletAddress, courseName) {
  const amount = ethers.parseUnits('250', 18); // 250 KENO
  
  const tx = await utl.distributeTokens(
    walletAddress,
    amount,
    `Completed: ${courseName}`
  );
  
  await tx.wait();
  console.log(`✅ Distributed 250 KENO to ${walletAddress}`);
  return tx.hash;
}

Staking Integration

Platforms can display users' staking balances and pending rewards from within their own UI — giving students a reason to stake and hold KENO instead of immediately spending it.

JavaScript — Read Staking Data 
const STAKING_ABI = [
  'function stakedBalance(address) view returns (uint256)',
  'function pendingRewards(address) view returns (uint256)',
  'function totalStaked() view returns (uint256)',
];

const staking = new ethers.Contract(
  '0x49961979c93f43f823BB3593b207724194019d1d',
  STAKING_ABI,
  provider
);

// Display in your user dashboard
async function getUserStakingInfo(walletAddress) {
  const [staked, rewards] = await Promise.all([
    staking.stakedBalance(walletAddress),
    staking.pendingRewards(walletAddress),
  ]);
  return {
    staked:  ethers.formatUnits(staked, 18),   // e.g. "500.0"
    rewards: ethers.formatUnits(rewards, 18),  // e.g. "12.5"
  };
}

Fee Structure

Every transaction routed through UTL Protocol generates a small fee collected by the FeeCollector contract. This creates a self-sustaining revenue stream that flows back to KENO stakers and the protocol treasury.

Transaction Type Fee Destination
Token Distribution 1% FeeCollector → Staking Rewards Pool
KUTL Card Settlement 0.5% FeeCollector → Treasury
Staking Deposit/Withdrawal 0.25% FeeCollector → Staking Rewards Pool
Flash Arbitrage Loans (FAL™) 0.09% FeeCollector → KENO Holders
Third-party Platform Distributions 1% FeeCollector → Staking Pool + T.D.I.R. Foundation
The Toll Road Model

You built the road. Every platform that drives on it pays a toll. Every toll flows back to KENO holders. The more platforms that integrate UTL Protocol, the more fee revenue the ecosystem generates — for everyone.
